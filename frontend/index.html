➕ Nouvelle ligne

⚡ Formule SQL —

Cette formule remplace le RECHERCHEV — calculée directement dans PostgreSQL sans dépendre du chargement du Sheets.

Fonctions : pfn() (nombre fr → float), pfd() (date fr → date), toutes les fonctions PostgreSQL.
Mots-clés interdits : INSERT, UPDATE, DELETE, DROP, etc.

💡 Les deux types de formules

📐 Formule cellule (type Google Sheets)
Double-cliquez une cellule, tapez la formule en commençant par =, puis Entrée.
~400 fonctions disponibles, en français ou en anglais.
Le résultat est calculé instantanément côté navigateur.
=SOMME(palettes_entree) — somme de la colonne
=MOYENNE(B1:B10) — moyenne des 10 premières lignes de B
=SI(A1>100;"GROS";"petit") — condition
=NB.SI(client;"BBA Emballages") — comptage conditionnel
=RECHERCHEV("MAT";code:productivite;7;FAUX) — RECHERCHEV native
=A1*1,2 — arithmétique
⚠️ Voit uniquement les lignes de la page courante.
⚡ Formule SQL (colonne entière)
Bouton + Colonne ou Modifier formule.
Appliquée par PostgreSQL sur toutes les lignes (même 370 000), résultat persisté en BD.
Idéal pour les RECHERCHEV massifs sur de grosses tables.
pfn(palettes_entree) + pfn(pal_livree_sortie)
(SELECT SUM(pfn(s.duree_nbr)) FROM suivi_equipe s WHERE s.n_bl_n_palette = recap_bl.n__bl)

📖 API mhp — référence

Tous les scripts commencent par import mhp. La lib est disponible automatiquement.

📊 Tables
t = mhp.table('stock_it')
t.columns                              # liste des colonnes
t.rows(limit=100)                       # liste de dicts
t.rows(where='client = %s', params=['BBA'])
t.count(where="date LIKE '2025%'")
t.append_row({'date':'2026-01-01', 'palettes_entree':'12'})
t.append_rows([{...}, {...}, {...}])    # batch insert
t.update_cell(primary_val='2026-01-01', column='client', value='ABC')
t.delete_row(primary_val='2026-01-01')

mhp.tables()                            # liste toutes les tables
mhp.sql("SELECT date, COUNT(*) FROM stock_it GROUP BY date")

⏱ Triggers automatiques
▶ Manuel       — déclenché par le bouton ▶
⏱ Cron         — déclenché à des horaires (ex: '0 8 * * *' = 8h tous les jours)
                 timezone Europe/Paris
✏️ on_edit     — déclenché à chaque modification de cellule
                 contexte fourni via os.environ :
                   MHP_TRIGGER_TABLE, MHP_TRIGGER_COLUMN,
                   MHP_TRIGGER_PRIMARY_VAL, MHP_TRIGGER_OLD_VALUE,
                   MHP_TRIGGER_NEW_VALUE
➕ on_row_add  — déclenché après un INSERT
                 contexte : MHP_TRIGGER_TABLE, MHP_TRIGGER_PRIMARY_VAL,
                   MHP_TRIGGER_ROW_DATA (JSON)

Exemple on_edit :
  import os, mhp
  if os.environ['MHP_TRIGGER_COLUMN'] == 'client':
      mhp.log(f"Client modifié : {os.environ['MHP_TRIGGER_OLD_VALUE']}"
              f" → {os.environ['MHP_TRIGGER_NEW_VALUE']}")
🛡 Sandbox (RestrictedPython)

Quand activé : le code est compilé via RestrictedPython, qui bloque les imports dangereux (os.system, subprocess, eval, etc.). À utiliser pour les scripts écrits par des utilisateurs moins fiables.

Inconvénient : limite les librairies utilisables. La lib mhp reste disponible.

⚙️ Limites
  • Timeout : 60 secondes par exécution
  • Sortie capturée : 100 000 caractères max
  • Le script tourne avec les droits de la BD : peut tout lire/écrire
  • Bibliothèques : tout ce qui est dans le venv backend (psycopg2 inclus)
